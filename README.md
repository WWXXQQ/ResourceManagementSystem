## 快速启动

### 环境要求

- Python 3.10+
- pip

### 本地运行

```bash
# 1. 创建虚拟环境
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Linux/Mac

# 2. 安装依赖
pip install -r requirements.txt

# 3. 初始化数据库
python manage.py migrate

# 4. 创建管理员
python manage.py createsuperuser

# 5. 启动服务
python manage.py runserver
```

访问地址：
- 前台：`http://127.0.0.1:8000/`
- 后台管理：`http://127.0.0.1:8000/admin/`

> 生产部署请参阅 [deployment_guide.md](deployment_guide.md)

## 系统初始化

首次使用需在**后台管理**中按以下顺序配置：

| 步骤 | 后台菜单 | 说明 |
|:---:|:---|:---|
| ① | 下拉选项配置 | 配置团队、卡型号、卡形态、受益项目、地域 |
| ② | 用户管理 | 创建/批量导入用户，分配角色和团队 |
| ③ | 卡池库存 | 按型号+形态+地域创建库存池，设定总量 |
| ④ | 申请单列表 | 可选：批量导入历史申请单 |
| ⑤ | 物料资产 | 可选：录入/批量导入服务器台账 |

### 批量导入格式

**用户导入**（Tab 分隔）：
```
用户名	团队	账号(可选)
张三	决策智能	zhangsan
```
- 初始密码统一为 `123456`，用户首次登录后自行修改

**物料资产导入**（Tab 分隔，从 Excel 直接复制粘贴）：
```
资源名称	IP地址	密码	规格	卡型号	卡形态	地域	状态	责任人	使用人	应用
```

## 审批流程

```
申请人提交 → 组长审批 → 预审人预分配 → 部门负责人终审 → 执行人执行 → 释放
```

```
PENDING_TEAM → PENDING_PRE → PENDING_FINAL → APPROVED → EXECUTED → RELEASED
                                                         ↗ (驳回)
任意审批节点均可驳回 ──────────────────────────→ REJECTED
```

### 撤回机制

在下一个角色处理之前，审批人可以**撤回**自己的审批结果：

| 操作人 | 可撤回条件 | 效果 |
|:---|:---|:---|
| 组长 | 申请仍在"待预审" | 回到"待组长审批"，可重新审批 |
| 预审人 | 申请仍在"待终审" | 回到"待预审"，清空预分配数据 |

撤回入口位于**已审批记录**选项卡的「操作」列。

## 各角色操作指南

### 申请人

1. **提交申请**：「资源申请」→ 填写项目、卡型号/形态、数量、使用人等 → 提交
2. **查看进度**：「资源看板」仪表盘显示申请状态与通知
3. **附件上传**：支持选择文件或 **Ctrl+V 直接粘贴图片**
4. **修改密码**：右上角「修改密码」

### 组长

1. **审批**：「待办审批」→ 按项目分组查看本团队申请 → 逐条同意/驳回
2. **批量同意**：点击项目组右上角「批量同意」一键通过整个项目
3. **撤回**：切换到「已审批记录」→ 对"待预审"状态的单据点击「撤回」

### 预审人/审批人

1. **预分配**：查看库存参考面板 → 填写预分配数量/型号/形态 → 保存并送交终审
2. **批量预分配**：库存充足时可一键批量预分配整个项目（库存不足时需逐条处理）
3. **紧急抽调**：库存不足时，底部出现 🔄 资源协调抽调面板 → 从已分配项目中指定抽调数量
4. **撤回**：已审批记录中对"待终审"单据点击「撤回」重新分配

### 部门负责人

1. **终审**：查看预分配方案（含抽调计划）→ 批准或驳回
2. **批量终审**：点击项目组「批量终审通过」
3. 终审通过后系统自动扣减库存

### 执行人

1. **确认执行**：「待办执行」→ 查看已批准的申请 → 填写执行结果 → 标记已执行
2. **绑定物料**：裸机类型可绑定物料资产到申请单
3. **释放资源**：到期后释放资源，系统自动归还库存
4. **物料管理**：增删改查服务器台账（IP、密码、卡数、规格等）

### 管理员

- 拥有所有角色的权限
- 通过 `/admin/` 后台进行系统配置、用户管理、数据导入
- 查看所有申请单和审批状态

## 核心功能说明

### 库存管理

- **卡池库存**按 `卡型号 + 卡形态 + 地域` 组合管理
- 预审阶段显示**安全余量**（= 总量 - 已分配 - 待终审预锁定）
- 终审通过时正式扣减库存，释放时自动归还

### 物料资产

- 管理物理服务器/设备台账：名称、IP、密码（加密存储）、规格
- **卡数量**字段：默认 8 卡/台，支持按实际配置修改
- 密码通过 AES 对称加密存储，仅执行人/管理员可查看
- 支持 CSV 导出

### 紧急资源抽调

当可用库存为 0 时，预审人可从已批准/已执行的工单中协调抽调资源：

1. 系统自动列出同型号+形态的候选工单
2. 预审人填写每个候选的拟抽调数量
3. 方案附带在预审意见中送交终审
4. 终审通过后由执行人完成实际资源调配

### 智能催办

- 超过阈值天数未处理的申请会显示催办提醒
- 支持配置外部 Webhook（飞书/钉钉/企微），详见 [deployment_guide.md](deployment_guide.md)

### 问题反馈

- 所有角色可通过「问题反馈」提交 Bug/建议
- 支持多图上传和 **Ctrl+V 粘贴截图**
- 管理员在后台查看和管理反馈

## 项目结构

```
ResourceManagementSystem/
├── gpu_management/          # Django 项目配置 (settings, urls, wsgi)
├── resource_app/            # 主应用
│   ├── models.py            # 数据模型 (用户/库存/申请/资产/反馈等)
│   ├── views.py             # 视图逻辑
│   ├── admin.py             # 后台管理配置 (含批量导入)
│   ├── urls.py              # URL 路由
│   └── templates/           # HTML 模板
│       ├── base.html        # 基础布局 (侧边栏/顶栏)
│       ├── login.html       # 登录页
│       ├── dashboard.html   # 资源看板
│       ├── apply.html       # 资源申请
│       ├── approve.html     # 审批管理
│       ├── execute.html     # 执行管理
│       ├── assets.html      # 物料管理
│       ├── statistics.html  # 累计统计
│       └── feedback.html    # 问题反馈
├── deployment_guide.md      # 生产部署指南
├── data_migration_guide.md  # 数据迁移指南
├── requirements.txt         # Python 依赖
├── manage.py                # Django 管理命令
└── start.bat                # Windows 一键启动脚本
```
